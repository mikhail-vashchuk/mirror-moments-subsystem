# Mirror Moments Subsystem

A C++ OOP study-driven mini-project focused on the analysis, design, and early implementation of a subsystem for creating and joining live moments in Mirror.

## Overview

Mirror is a concept of a digital presence space built around live moments between people.

In Mirror, a person does not simply start a stream or open a room. They launch a moment: a meaningful live situation, experience, or interaction that other people can discover, observe, and request to join.

This repository focuses on one specific part of that idea: the subsystem for creating live moments and joining them.

## Project goals

- practice object-oriented analysis and design in C++
- turn academic work into a coherent mini-project
- model a meaningful real-world concept
- build a clean foundation for possible future development of Mirror

## Core idea

The subsystem is centered around several key concepts:

- a user can create a live moment
- other users can discover moments
- a user can request to join a moment
- the host can accept or reject the request
- a live moment can later become part of the user's reflection

The goal is not to model a standard chat or social feed, but to design a structured system for live, meaningful, joinable moments between people.

## Planned development stages

This project is developed through a sequence of object-oriented academic stages:

1. domain analysis and problem statement
2. object hierarchy
3. class hierarchy
4. object interactions
5. object state transitions
6. class definitions in C++
7. class implementation in C++
